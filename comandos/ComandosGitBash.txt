Comandos Git:

git init . -> torna a pasta especial, possibilitando versionamento com git

git status ->
Consegue verificar arquivos que ainda não estão participando do versionamento

git add -> prepara o que vai "commitar"

git config --global user.email "kaiquepfreitas@gmail.com"
configurar e-mail que está registrando aquelas alterações
isso faz-se apenas uma vez - configuração "global"
Para testar se deu certo -> git config --global user.email / git config --global user.name / aparecerá nome e e-mail cadastrado

git commit -m "Primeiros comandos com bash"

git log -> histórico de todos os commits

"." -> pasta atual

git add "." -> commita tudo da pasta atual

git reset "nome do arquivo+extensao"
remover arquivo da fase de sague, quando utiliza o comando git add

git checkout -b +"nome branch - a ser definido"
obs.: a branch secundaria é criada a partir da branch atual. Então se estiver em uma branch que seja diferente da main, os arquivos clonados da branch atual e não da main.

git branch -> mostra todas as branchs do projeto e sinaliza a branch atual

git checkout main -> volta para a branch principal (main)

git merge + "Nome_Branch" -> junta os commits da branch secundaria na branch principal - apenas comandos "commitados"

git branch -D "nome da branch a ser excluída" -> uma branch
para que o comando funcione, você não pode estar na branch que está sendo excluída

git pull origin main -> atualizar o repositório offline e deixando igual ao repositório remoto - caso de ter saído alguma atualização por parte de outras pessoas do time, você atualiza o projeto local

git push -u origin nomeDaBranch -> subir alterações para repositório no GitHub

##CONFLITOS
Quando faz o merge de duas branchs secundarias distintas e que estão alterando a mesma linha, por exemplo, no caso de arquivos de texto. Cria-se uma etapa main-merging, onde o arquivo com conflito deve ser acessado e estarão destacadas todas os conflitos - devendo ser alterado manualmente, para corrigir, e commitar essa alteração. Após isso encerra-se a etapa main-merging e automaticamente o local é alterado para a branch main, que estava tendo conflito.
